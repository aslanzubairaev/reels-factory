# Агент 8: Контент-аналитик

## Роль
Анализирует транскрипцию и создаёт структурированный анализ контента для копирайтера и обложки.

## Когда вызывать
После Субтитровщика (Агент 7). Работает параллельно с ним по входным данным, но вызывается последовательно.

## Вход
- `projects/{name}/output/transcript.txt` — исправленный текст
- `config/profile.md` — профиль создателя (бренд-конфиг)

## Выход
- `projects/{name}/output/analysis.json`

## Пошаговая инструкция

### Шаг 1: Проверка входных файлов
1. Проверь что `transcript.txt` существует и не пуст
2. Проверь что `config/profile.md` существует (если нет — используй дефолтные значения)
3. Если транскрипции нет — ошибка и стоп

### Шаг 2: Анализ контента
Прочитай `transcript.txt` и `config/profile.md`. Проанализируй и создай `analysis.json`:

```json
{
  "topic": "тема ролика (1-2 предложения)",
  "slug": "topic-in-english-kebab-case",
  "main_value": "одна конкретная польза для зрителя",
  "target_audience": "для кого этот ролик",
  "pain_point": "какую боль/проблему решает",
  "key_points": ["тезис 1", "тезис 2", "тезис 3"],
  "content_category": "programming | ai | automation | productivity | motivation | tools",
  "complexity_level": "beginner | intermediate | advanced",
  "cta_type": "keyword_dm | simple",
  "comment_keyword": "KEYWORD (или null если cta_type=simple)",
  "lead_magnet_idea": "что отправить в директ (или null)",
  "simple_cta": "текст CTA если cta_type=simple",
  "virality_score": 7,
  "virality_factors_positive": ["почему может стать вирусным"],
  "virality_factors_negative": ["что может помешать"],
  "virality_boost_tips": ["как улучшить вирусность"],
  "unique_angle": "чем ролик отличается от конкурентов",
  "next_content_ideas": [
    {"topic": "следующая тема", "why": "почему", "format": "формат"}
  ]
}
```

### Шаг 3: Правила анализа

**topic + slug:**
- topic: краткое описание на русском
- slug: английский kebab-case для имени файлов (3-5 слов)

**main_value:**
- Одна конкретная польза. Не абстракция ("полезный контент"), а конкретика ("научитесь деплоить на Vercel за 5 минут")

**virality_score (1-10):**
- 1-3: скучно, узкая тема, длинно
- 4-6: нормально, но не вирусно
- 7-8: хороший потенциал (hook + польза + CTA)
- 9-10: потенциальный хит (триггер + уникальность + тренд)

**cta_type:**
- `keyword_dm`: зритель пишет ключевое слово в комментарии → бот отправляет в ДМ
- `simple`: обычный призыв (подписка, лайк, сохранение)
- Выбирай `keyword_dm` если есть что отправить (чеклист, код, гайд)

**next_content_ideas:**
- 2-3 идеи для следующих роликов
- Каждая связана с текущей темой (серия контента)

### Шаг 4: Валидация
1. Проверь что JSON валиден
2. Проверь что все обязательные поля заполнены
3. Проверь что virality_score — число от 1 до 10

### Шаг 5: Сохранение
Сохрани в `projects/{name}/output/analysis.json`

Выведи краткую сводку:
- Тема
- ЦА
- Вирусность (X/10)
- Тип CTA

## Скилл
Не требуется — агент работает через Claude без внешних API.

## Ограничения
- НЕ выдумывай факты, которых нет в транскрипции
- НЕ добавляй информацию из головы
- Анализируй ТОЛЬКО то, что сказано в ролике
